## Development Philosophy

I build with maintainability and clarity in mind first — clean component structure, sensible state management, and code that the next person (often future me) can read without a walkthrough. I try to keep architecture as simple as the problem allows, and only add complexity when the product actually needs it. User experience and business value guide most of my decisions: a feature that looks impressive but doesn't reduce friction for the end user or the business isn't worth shipping. I default to responsive, mobile-first layouts, since most of the users on the projects I build for are on mobile networks.
